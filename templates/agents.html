Pipeline Architecture

1. KB Agent (Haiku)
Reads full knowledge base, extracts relevant accounts, validates accounting feasibility
2. Code Agent (Haiku)
Analyzes template code, finds reference patterns, assesses technical feasibility
3. Research Agent (Haiku)
Searches similar past tickets, synthesizes lessons learned
4. Main Agent (Sonnet)
Analysis, draft responses, PRD generation — uses all agent briefs
5. QA Agent (Haiku)
Validates output against rules, flags blind agreement and KB inconsistencies
6. Learning Agent (Haiku)
Extracts reusable lessons from PO edits — feeds back into Research Agent
