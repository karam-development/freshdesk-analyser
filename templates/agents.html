Pipeline Architecture

All 16 registered agents — active agents run today, reserved agents are wired for future use. No agent auto-sends replies — all output requires human review.

Active — in use now

1. main_analysis_agent (Sonnet · ticket analysis)
Classifies ticket, produces structured analysis, RICE scores, PM decision context. Entry point of the pipeline.
2. kb_agent (Haiku · parallel prep)
Reads knowledge base, extracts relevant entries, validates accounting feasibility for the draft.
3. code_agent (Haiku · parallel prep)
Analyses template code, finds reference patterns, assesses technical feasibility. Output is plain-language — no raw code.
4. research_agent (Haiku · parallel prep)
Searches past tickets for similar resolved issues and patterns. Surfaces precedents and lessons for the draft.
5. draft_response_agent (Sonnet · draft generation)
Generates FR and EN draft responses using all agent briefs, KB evidence, PM constraints, and structured lessons.
6. qa_agent (Haiku · QA)
Reviews draft for quality, tone, completeness, and PM-constraint compliance. Flags issues. May trigger one regeneration cycle.
7. learning_agent (Haiku · learning loop)
Extracts reusable lessons from PO edits and Freshdesk replies. Stores them as structured PM lessons for future drafts.
8. prd_agent (Sonnet · PRD generation)
Generates Product Requirements Documents for feature requests and complex changes requiring structured specification.

Reserved — registered, not yet wired

classification_agent Haiku · fast pre-classification before full analysis to reduce main agent token usage
summary_agent Haiku · compact ticket summaries for dashboard previews and weekly digests
feasibility_agent Haiku · dedicated technical feasibility pre-screen, separate from code agent
batch_agent Haiku · scheduled batch jobs: nightly re-analysis, bulk KB refresh, re-scoring
reply_scanner_agent Haiku · scans incoming Freshdesk replies for PO corrections and lesson signals
jira_agent Haiku · searches and summarises related Jira issues to surface known bugs and in-flight work
notification_agent Haiku · internal notifications (Slack, email) on approval, flags, and escalations
reporting_agent Sonnet · weekly/monthly reports: ticket volumes, resolution rates, PM decision breakdown
